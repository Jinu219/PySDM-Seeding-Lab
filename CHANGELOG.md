# Changelog

All notable project changes are documented in this file. The project follows
semantic versioning from the v1.0 release onward.

## [1.0.0] - 2026-07-21

### Added

- Reproducible single, control-versus-seeding, sweep, and ensemble workflows using
  the real PySDM parcel adapter.
- Native warm-cloud diagnostic products, wet-radius spectra, water budgets,
  numerical convergence audits, threshold robustness, and publication reports.
- Common-seed response qualification with resumable execution, durable evidence,
  process-isolation diagnostics, and bounded server workers.
- Streamlit experiment design, execution, result exploration, server-job monitoring,
  artifact downloads, and legacy-result compatibility.
- Observation-event validation with immutable source hashes, synthetic-evidence
  separation, BASTALIAS moving-platform import, and ARM ENA fixed-column import.
- Cross-platform Windows/Ubuntu fast CI, real PySDM Ubuntu integration, a finite
  release gate, and machine-checked scientific claim scope.

### Scientific scope

- Absolute rain-water state is supported only for the reviewed collision-ON
  targeted marine parcel profile and its tested resolution comparisons.
- Positive finest-reference response direction is descriptive only.
- The 1% transition floor is operational only and requires documented threshold and
  cadence sensitivity.
- Quantitative seeding response, external observational calibration, and real-world
  field efficacy are explicitly unsupported in v1.0.

### Deferred

- Direct Lagrangian parcel observation calibration remains future research.
- Serial versus 4/8-worker real-PySDM server scaling is deferred to v1.1.
